# 🎪 展位销售与财务综合管理系统 (Pro版)

基于 **Cloudflare Workers + D1 + R2** 架构构建的新一代展会垂直管理系统。系统深度整合了招展业务流与财务合规流，旨在消除数据孤岛，实现展位售价、财务流水与成本支出的实时联动。

---

## 🚀 核心功能模块

### 1. 动态展位库与 BI 大盘
* **单价联动机制**：展位单价不再是静态死数。成交前根据配置显示“原价”或“定制价”；成交后自动关联订单金额，实时转化为“实际单价”。
* **多维联动大盘**：数据概览支持按 **展馆、展位类型、销售进度** 进行实时筛选统计。看板数据随列表筛选条件即时刷新，实现精准的预期收入核算。
* **状态强锁定**：已预订/已成交展位自动进入只读锁定状态，操作栏显示锁定图标，防止底层资产数据被误改。

### 2. 精准订单录入系统 (UX 增强版)
* **招展渠道合规**：强制区分“直招”与“代理商招展”。选择代理商时激活必填验证，确保渠道来源可追溯。
* **产品分类与行业字典**：支持管理员预设“产品分类”字典。录入员通过下拉联想快速归类，同时保留“详细展品”文本框供自由描述。
* **智能地区选择**：省份下拉框按拼音首字母 (A-Z) 分组排列；针对“国际”地区开启特定国家名称的强制必填校验。

### 3. 财务三权分立体系
系统将财务操作解耦为三个专业频道，确保权责清晰：
* **💰 收款流水管理**：支持预设项目专属对公账户，录入时一键勾选，并兼容微信、支付宝、现金等快捷收款记录。
* **🛠️ 费用信息变更**：支持在不退单的前提下，对成交价及结构化杂费进行动态修正，系统实时重算全款/尾款状态。
* **📤 代付与返佣 (含事由)**：
    * **侵蚀逻辑**：支持直接从订单已收款总额中抵扣支出，系统自动校验支出比例。
    * **事由追溯**：每一笔支出必须填写明确事由。
    * **专业单据打印**：一键生成排版精美的《项目款项支付/返佣申请单》，含申请人信息、收款详情及审批签字位。

### 4. 数据档案与归档
* **全景档案编辑**：点击客户名进入档案中心，查看包括录入人员、录入时间在内的全量明细，并支持编辑非关键性业务资料。
* **一键全量导出**：支持导出 CSV 格式大盘数据，涵盖所有财务明细、业务员归属及云端合同下载链接。

## 🛠️ 技术栈

* **Runtime**: Cloudflare Workers (V8 Engine)
* **Database**: Cloudflare D1 (SQL)
* **Storage**: Cloudflare R2 (Object Storage)
* **UI Framework**: Tailwind CSS 3 (CDN版)
* **Logic**: Vanilla JavaScript (原生脚本，零框架依赖)

---

## 📦 部署与配置要点

### 1. 数据库准备
在 Cloudflare D1 控制台需确保以下核心表已建立并完成字段扩容：
* `Staff` / `Projects` / `Project_Staff_Map`
* `Booths` / `Project_Prices`
* `Orders` (需包含 `category`, `profile`, `fees_json` 字段)
* `Payments` (收款流水)
* `Expenses` (需包含 `reason`, `payee_channel` 字段)
* `Project_Accounts` / `Project_Industries` (配置字典)

### 2. 初始化建议顺序
1.  **创建项目**：设定项目年份及展期。
2.  **定价策略**：设定项目各展位类型的全局单价。
3.  **账户配置**：在【系统配置】中录入对公收款账户信息。
4.  **行业字典**：配置“产品分类”下拉选项。
5.  **展位导入**：通过 Excel 批量导入展位基础物理信息。

## 📌 维护说明

* **初始密码**：系统默认密码为 `123456`，管理员应要求业务员首次登录后立即通过侧边栏修改。
* **数据一致性**：由于采用强关联查询逻辑，建议不要在数据库后端直接物理删除已成交订单关联的展位记录。
* **协作工作流**：接管后的协作方式见 [docs/WORKFLOW.md](./docs/WORKFLOW.md)。
* **首次 Git 提交**：第一次把本地项目接入 GitHub 的流程见 [docs/GIT-FIRST-COMMIT.md](./docs/GIT-FIRST-COMMIT.md)。
* **D1 变更留档**：以后所有数据库改动统一存放在 [migrations/README.md](./migrations/README.md) 对应的 `migrations/` 目录。
