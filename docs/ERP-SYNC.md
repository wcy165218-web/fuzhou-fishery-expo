# ERP 收款同步说明

## 现在的实现

系统已经支持把 ERP 中“已认领完成”的收款，按企业名称匹配到当前项目订单，并自动写入本系统的 `Payments` 流水。

支持的能力：

- 管理员在“系统与人员配置”页面里保存 ERP 配置
- 先做预检查，再正式同步
- 自动去重，不会重复导入同一条 ERP 记录
- 自动拦截超出订单应收总额的同步
- ERP 导入的收款流水会标记为只读，前端不能手改或手删
- 按分页自动抓取 ERP 数据，不只读取第一页

## JSESSIONID 存在哪里

`JSESSIONID` 不保存在代码文件里。

它保存在数据库表 `ProjectErpConfigs.session_cookie` 中，对应的是你当前项目的 ERP 同步配置。

也就是说：

- 你不用改代码文件
- 你不用改 `wrangler.toml`
- 你不用改 GitHub
- 只需要在系统页面里重新粘贴并保存

## 手动更新 JSESSIONID 的位置

路径：

1. 登录系统
2. 进入“系统与人员配置”
3. 找到“ERP 收款同步配置”
4. 把新的 `JSESSIONID` 或完整 `Cookie` 粘贴到 `JSESSIONID / Cookie` 输入框
5. 点击“保存 ERP 配置”

## 什么时候需要更新 JSESSIONID

这个时间不是我们系统决定的，而是 ERP 自己的登录会话策略决定的。

所以没法提前精确说“一定几小时”或“一定几天”。

通常你在这些情况更新就行：

- ERP 退出登录后
- ERP 长时间没操作后
- 浏览器重新登录 ERP 后
- 本系统点击“先做预检查”或“正式同步入账”时报错提示 ERP 认证失效

最简单的理解：

只要 ERP 会话没过期，就继续能用；一旦过期，再复制一次新的 `JSESSIONID` 即可。

## 怎么重新拿到 JSESSIONID

路径：

1. 打开 ERP 的收款认领结果页面
2. 按 `F12`
3. 打开 `Network`
4. 找到 `hyDailyWaterSubController.do?datagrid...`
5. 点开 `Headers`
6. 在 `Request Headers` 里找到 `Cookie`
7. 复制最新的 `JSESSIONID=...`

如果方便，也可以直接复制整段 `Cookie`，系统同样支持。

## 能不能自动更新

可以尝试，但前提是 ERP 本身要允许。

### 方案 1：继续手动更新

这是现在最稳、最快上线的方案。

优点：

- 不需要动 ERP
- 不需要账号密码写进程序
- 现在就能用

缺点：

- 会话过期后要手动重新复制一次

### 方案 2：程序自动登录 ERP

这个是理论上最理想的方案，但要满足下面条件：

- ERP 有明确可调用的登录接口
- 登录不需要验证码 / 短信 / 动态校验
- 你能提供登录接口抓包信息
- 可以接受把 ERP 登录凭证保存到系统里

如果满足这些条件，我就可以继续做“自动登录 ERP -> 自动拿新 JSESSIONID -> 再同步收款”。

### 方案 3：ERP 提供正式接口 Token

如果对方 ERP 能给固定 API 账号、Token、签名方式，这是最专业也最稳定的方式。

但这取决于 ERP 厂商是否支持，不是我们自己能单方面决定的。

## 我目前的建议

先上线手动更新版。

原因：

- 现在已经能工作
- 风险最低
- 你只在会话失效时更新一次，不需要每次同步都改

等后面你拿到 ERP 登录接口抓包，我再继续升级成自动更新版。
